Segue to Dialogflow

Dialogflow is a service that...



"gives users new ways to interact with your product by building engaging voice and text-based conversational interfaces, such as voice apps and chatbots, powered by AI."

How does it work?

  • Text or speech → Maps to intents → Match and extract parameters/entities → Trigger fulfillment → Deliver response via integration
  • Leverages NLP/machine learning in intent matching and parameter extraction

Why you should care




Easy to use

Solves hard problems

Its a service, not a platform

Many built-in integrations

  • Easy to use - Abstractions that make sense and the tools are approachable
  • Solves hard problems - NLP and Machine Learning requires specialized skillset and even then is very difficult
  • It's a service, not a product or platform - It does what it does well and gets out of your way
  • Out of the box integrations - Works with many popular chat/voice platforms such as Slack, Google Assistant, Facebook Messenger, Google Hangouts, Skype, Twilio, Google Telephony, and more...
  • Conversations as a Service

Dialogflow Concepts

Agent

  • Virtual agent that handles conversations with your end users
  • Similar to a human call center agent
  • You train your agent to handle conversation scenarios
  • Top-level container for settings

Intents


Categorizes the user's intention for a single conversation cycle


  • Similar to Alexa's intent
  • Can (and should) have many intents
  • User text or speech is matched against an intent (by Dialogflow)
  • Output of an intent is a response to the user
  • I think of an intent as a method call for your agent

Training Phrases



Order a pizza
Can I get a small pizza, please?
I'd like a cheese pizza
Send me a pan pizza
Deliver a medium thin crust pizza
Order a large pizza with sausage, pepperoni and cheese
Send me a pie!

  • Example phrases of what a user might say
  • Used to match an intent
  • You don't need to provide every example but 10 training phrases is a best practice

Default Intents



Welcome intent

Training phrases: hi, hey, howdy, hi there, what's up?, greetings, long time no see
Responses: Hi there, Can I help you?, What can I do for you?



Fallback intent

Responses: Can you repeat that?, Sorry?, What was that?, Does not compute

  • Created when an agent is created
  • Welcome - triggered by the welcome event
  • Fallback - triggered when no custom intent is matched

Parameters



Can I get a small pizza, please?
I'd like a cheese pizza
Send me a pan pizza
Deliver a medium thin crust pizza
Order a large pizza with sausage, pepperoni and cheese
Deliver me an xl, pan pizza with spinach and onions

  • The parts of the user utterance that are important for building a response
  • Each intent has it's own set of parameters (you can have more than 1)
  • Just like method parameters

Define and annotate


  • Name - unique name
  • Entity - type (more on this later)
  • Value - Usually same as name
  • Required? - Parameter must be provided
  • Is List? - Multi-item parameters
  • Prompts? - Used when required parameter is missing
  • Default value - Value provided for missing, non-required params
  • Dialogflow console provides a nice GUI for annotation
  • Define parameters before training phrases

System entities

  • Types for your parameters - determines how parameter values are extracted
  • System entities = built-in
  • Developer entities = custom. Powerful

System Entities

Date and Time date-time, date, date-period, time, time-period

Amounts

  • Create your own entities to supplement system entities
  • Type: type of information you want to extract (e.g. vegetable)

  • Aliasas: equivalent phrases for each enty

  • API to update entries

Fulfillment


Code that executes
in response to an intent
to perform custom logic and responses

  • Code that could execute in response to an intent
  • Provide dynamic logic and/or responses
  • Only needed if you are using 1 or more standard integrations

How it works

  • Intent is matched
  • If fulfillment is enabled for intent
  • Triggers fulfillment with name of intent, parmaters, etc..
  • Fulfillment performs logic and creates response

Inline editor

  • Deployed as a Firebase function
  • Limitations:
  • Node 8
  • Only supports package.json and index.js
  • Deployment can be slow
  • Requires billing to be enabled (in Firebase) for network calls outside Google network (5gb free/mo and $.12 per/gb)

Custom webhook

  • Publicly addressable
  • Supports HTTPS
  • Secure by Basic auth or custom header

NodeJS Library



const functions = require('firebase-functions');
const { WebhookClient } = require('dialogflow-fulfillment');

	const agent = new WebhookClient({ request, response });

Context


User: What's the weather like in Richmond, VA?
Agent: Right now in Richmond, it's 78° under partly cloudy skies.


User: Will it rain tomorrow?
Agent: Tomorrow in Richmond, there is an 80% chance of rain.


User: Will it be hot?
Agent: Tomorrow in Richmond, the high is 85° and the low is 66°.

  • Just like a normal conversation context
  • Your users don't have to repeat themselves
  • You can control the flow of the conversation
  • Make the interaction feel more natural
  • Defined as a "string" with some associated data

How it works

  • If an intent has an output context, it becomes active whenever that intent is matched
  • If an intent has an input context, it only matches if that context is active
  • Lifespan - defines how the duration of the context (time and turns)
